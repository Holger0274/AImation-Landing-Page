## Project Overview

**AI.mation Landing Page** - A landing page for AI automation consultancy targeting German SMEs (small and medium enterprises).

- **Brand**: AI.mation
- **Claim**: Automatisierung mit Intelligenz
- **Owner**: Holger (20+ years engineering leadership experience)
- **Target Audience**: KMU decision makers in DACH region (10-1000 employees) - Geschäftsführer, Bereichsleiter, Abteilungsleiter
- **Target Market**: Small and medium enterprises - produzierendes Gewerbe, Dienstleister, größere Handwerksbetriebe, mittelständische Industrieunternehmen
- **Language**: German (all content must be in German)
- **Tone**: Modern, competent, accessible, innovative - Professional but approachable, NEVER conservative or "bank-like", NO buzzwords
- **USP**: 20 Jahre Engineering-Erfahrung kombiniert mit tiefem Prozessverständnis
- **Pricing**: Auf Anfrage (prices on request)

### Target Personas

The landing page addresses three distinct personas with different needs:

**Persona A: Der Entscheider** (The Decision Maker)
- Role: Geschäftsführer, Inhaber, Bereichsleiter
- Company: 50-1000 employees
- Situation: Hears "AI" everywhere, feels pressure to act, no internal AI know-how
- Seeks: Orientation + someone who also implements
- Address with: ROI, competitive advantage, "Machen statt reden"

**Persona B: Der Umsetzer** (The Implementer)
- Role: Abteilungsleiter, Prozessverantwortlicher, IT-Leiter
- Situation: Tasked with exploring AI
- Seeks: Training for self/team + concrete use cases to show
- Address with: Practical examples, quick wins, "Das können Sie morgen anwenden"

**Persona C: Der Skeptiker** (The Skeptic)
- Role: Controller, Qualitätsleiter, critical shareholder
- Situation: Asks for numbers, not buzzwords
- Seeks: Consulting with clear business case
- Address with: ROI calculation, risk minimization, references

### Service Model: 3 Pillars

AI.mation offers services across three interconnected pillars:

1. **Säule 1: Schulungen** (Training) - Wissen vermitteln
   - 3 levels: Einstieg & Awareness, Anwendung & Tools, Fortgeschritten & Spezialisiert
   - Topics: Generative KI, Prompt Engineering, Microsoft Copilot, Multi-Agent-Systeme, Vibe Coding, etc.

2. **Säule 2: Beratung** (Consulting) - Denken, planen, entscheiden
   - 3 phases: Analyse (Wo stehen wir?), Strategie (Wo wollen wir hin?), Begleitung (Wie kommen wir dahin?)
   - Modules: AI Readiness, AI Audit, Use Case Identification, Roadmap, Change Management, Governance

3. **Säule 3: Umsetzung** (Implementation) - Bauen, implementieren, betreiben
   - 4 solution worlds: FLOW (Workflows), KNOW (Knowledge), THINK (Innovation), WORK (Assistants)
   - Concrete solutions: RAG systems, automation workflows, intelligent agents

**Key Message**: All services are modular and combinable. The three pillars work together perfectly: Consulting identifies use cases, training prepares employees, implementation builds solutions.

## Design System

This project follows a strict design system defined in `docs/aimation-design-system-v3-final.md`. **This design system must be followed precisely** - all visual elements, colors, typography, and components are strictly defined.

### Core Brand Colors (Critical)

**⚠️ IMPORTANT**: Navy Blue (#031d44) is NOT a brand color and must NEVER be used. Only colors from `docs/aimation-design-system-v3-final.md` are valid.

**Official Brand Colors**:
- **Magenta** `#f90093` - Primary brand color (10% usage) - ONLY for CTAs, buttons, clickable elements, and highlights
- **Soft Black** `#071013` - Text, headlines, dark sections (20% usage) - NEVER use pure black `#000000`
- **Warm White** `#faf9f7` - Main background (65% usage) - **NEVER use pure white `#ffffff` for page background**
- **Pure White** `#ffffff` - ONLY for cards, inputs, contrast elements
- **Light Blue** `#60AFFF` - Subtle accents only (5% usage) - NEVER for text or buttons

**Forbidden Colors**:
- ❌ Navy Blue `#031d44` - NOT a brand color, do not use
- ❌ Pure Black `#000000` - Use Soft Black `#071013` instead
- ❌ Any colors from service documents that are not in the design system

### Typography

- **Headlines/Buttons**: Space Grotesk (weights: 400, 500, 600, 700)
- **Body Text**: Inter (weights: 400, 500, 600)
- Google Fonts import required:
  ```html
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&family=Space+Grotesk:wght@400;500;600;700&display=swap" rel="stylesheet">
  ```

### Critical Design Rules

**ALWAYS:**
1. Use Warm White `#faf9f7` as page background, never pure white
2. Use gradient on Magenta buttons: `linear-gradient(135deg, #f90093, #ff4ecd)`
3. Apply glow effects ONLY on dark backgrounds (Soft Black sections)
4. One Magenta highlight word per headline (wrapped in `<span class="highlight">`)
5. Maximum ONE Magenta button per viewport (establishes clear visual hierarchy)

**NEVER:**
1. Use pure white `#ffffff` as page background
2. Apply glow effects on light backgrounds
3. Use Magenta for non-clickable elements
4. Use Light Blue for text (insufficient contrast)
5. Use multiple Magenta buttons in the same viewport
6. Use pure black `#000000` - always use Soft Black `#071013`
7. Use Space Grotesk for body text (headlines only)

### Signature Element: Glow Effects

Glow effects are a defining brand element, used ONLY on dark (Soft Black) backgrounds:

- **Button Glow**: `box-shadow: 0 0 30px rgba(249, 0, 147, 0.4)`
- **Text Glow**: `text-shadow: 0 0 40px rgba(249, 0, 147, 0.5)`
- **Background Glow**: Radial gradient with blur (60-80px)
- **Dot Glow**: `box-shadow: 0 0 10px #f90093`

### Button Hierarchy

1. **Primary (Gradient Magenta)** - Most important action (limit: 1 per viewport)
2. **Secondary (Gray Border)** - Alternative actions
3. **Ghost (White Border)** - For dark backgrounds
4. **Text Link (Magenta underline)** - Tertiary navigation

### Section Patterns

- **Light Hero**: Warm White background, Magenta CTA (no glow)
- **Dark Hero**: Soft Black background, glow effects, Magenta CTA with glow
- **Features**: Warm White/Gray-50 background, white cards with borders
- **Testimonial**: Pure White background
- **CTA Banner**: Soft Black background with glow
- **Footer**: Soft Black background

## Landing Page Structure

The landing page consists of 11 carefully orchestrated sections. Full technical specifications with animations and responsive behavior are in `aimation-landing-page-guideline.md`.

1. **Header**

   - Goal: Pattern interrupt - capture visitor in 3 seconds

   - Goal: Communicate service portfolio clearly
   - **Flip cards** for each pillar (3D flip on hover/tap)
   - Content from Beratung, Schulungen, Umsetzung docs

6. **Process Section**
   - Goal: Reduce complexity - "This is how easy it is"
   - 4-step timeline: Erstgespräch → Konzept → Umsetzung → Begleitung
   - Horizontal (desktop) / Vertical (mobile)
   - Draw animation for connecting lines

   - Goal: Build trust through third parties
   - Tool logos (make.com, n8n, Microsoft)
   - Animated counters: "20+ Jahre Erfahrung", "18.000+ LinkedIn-Follower"
   - Infinite scroll logo slider

8. **About Section**
   - Goal: Personal connection
   - Professional photo + authentic bio
   - Split layout: Image | Text
   - Reveal animation on image

9. **FAQ Section**
   - Goal: Address objections preemptively
   - Accordion style, 5-7 questions
   - Smooth expand/collapse

10. **Final CTA Section**
    - Goal: Last conversion opportunity
    - Dark background with glow effects
    - Two CTAs: Warm (book call) + Cold (download guide)
    - Magnetic buttons on desktop

11. **Footer**
    - Logo, contact info, LinkedIn
    - Legal: Impressum, Datenschutz
    - Copyright
