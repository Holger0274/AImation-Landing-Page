# Pillar-Page-Strategie für aimation.de

## Status Quo: Warum die Seite unsichtbar ist

**Problem:** aimation.de ist eine Single-Page-App. Google findet genau eine URL. Kein einziges Longtail-Keyword hat eine eigene Seite, die ranken kann.

**Ergebnis:** Bei Suchen wie "KI Beratung Mittelstand", "KI Agenten KMU", "KI Automatisierung Kosten" taucht aimation.de nicht auf. Stattdessen ranken:

| Wettbewerber | Stärke | Schwäche aus AImation-Sicht |
|---|---|---|
| ki-beratung-mittelstand.de | Exakte Domain, guter Content | Kein Engineering-Hintergrund |
| roover.de | Tiefgehende Pillar-Artikel (5.000+ Wörter) | Generisch, kein Fokus auf Agenten |
| kiba.berlin | Starker Blog, SEO-optimiert | Berlin-fokussiert |
| ki-beratung-deutschland.de | Gute Kostenübersicht, Unterseiten | Copy-Paste-Feeling, kein Differentiator |
| click-solutions.de | KI-Agenten-Fokus | Kein Engineering-Domain-Wissen |
| lucidlabs.de | Workshop + Agent in 6 Wochen | Enterprise-Pricing |
| kigazon.com | Österreich-Fokus, aktueller Content | Kein DACH-weiter Fokus |

**Dein unfairer Vorteil:** Keiner dieser Wettbewerber hat 20+ Jahre Engineering-Erfahrung in Konstruktion, QM, Fertigung, F&E. Keiner kann Use Cases wie "CNC-Sounds analysieren" oder "Lessons-Learned-Agent für Retrofit" liefern. Das ist dein Content-Moat.

## Pillar 1: KI-Schulungen für den Mittelstand

**URL:** `/ki-schulungen-mittelstand`
**Ziel-Keywords:** "KI Schulung Mittelstand", "KI Workshop KMU", "KI Weiterbildung Unternehmen", "Prompt Engineering Schulung"
**Suchintention:** Informational + Commercial ("Was gibt es? Was kostet das?")

### Seitenstruktur

1. **H1:** KI-Schulungen für den Mittelstand: Praxisnah, verständlich, sofort anwendbar
2. **Pain-Einstieg:** "Ihre Mitarbeiter nutzen ChatGPT heimlich auf dem Privathandy. Die Frage ist nicht ob, sondern wie kontrolliert."
3. **Drei Schulungsebenen:**
   - Einstieg: KI verstehen (halber Tag, für Geschäftsführung + Teamleiter)
   - Anwendung: KI-Tools souverän einsetzen (1 Tag, für operatives Team)
   - Fortgeschritten: Prompt Engineering + Automatisierung (2 Tage, für Power User)
4. **Konkreter Ablauf** jeder Schulung (nicht generisch, sondern mit Engineering-Beispielen)
6. **FAQ-Sektion** (featured snippet optimiert)
7. **CTA:** Erstgespräch buchen

### Interne Links
- Verlinkt auf: Landing Page, Beratungs-Pillar, Use Cases
- Verlinkt von: Blog-Cluster (Kosten-Artikel, Copilot-Vergleich, Prompt-Engineering)

### Differentiator-Content (was nur du liefern kannst)
- Beispiel-Prompts für Ingenieure: "Erstelle eine FMEA für [Bauteil] unter Berücksichtigung von [Prozess]"
- Schulungsinhalt: Wie man technische Zeichnungen als Datenquelle für KI nutzt

## Pillar 4: KI-Agenten für Unternehmen (Differentiator-Page)

**URL:** `/ki-agenten-unternehmen`
**Ziel-Keywords:** "KI Agenten Unternehmen", "autonome KI Agenten", "KI Agent vs Chatbot", "Multi-Agenten-System"
**Suchintention:** Informational ("Was ist das?") + Commercial ("Wer baut das?")

### Warum diese Pillar entscheidend ist
"KI-Agenten" ist DAS Trend-Thema 2026. Die IHK Regensburg schreibt darüber. BornCity berichtet. Jeder Wettbewerber positioniert sich. Aber keiner erklärt es aus der Engineering-Perspektive.

### Seitenstruktur

1. **H1:** KI-Agenten für Unternehmen: Was sie können, was sie kosten, ob Sie sie brauchen
2. **Erklärung in Klartext** (nicht "agentisch", sondern "autonom")
   - Was ist ein KI-Agent? (vs. Chatbot, vs. Automatisierung)
   - Die 7 Stufen der KI-Nutzung (Reifestufen-Modell)
   - Wann brauche ich einen Agenten, wann reicht Automatisierung?
3. **Konkrete Use Cases aus dem Engineering**
   - Lessons-Learned-Agent: Warnt automatisch bei bekannten Problemen
   - Qualitätssicherungs-Agent: Analysiert CNC-Sounds auf Anomalien
   - Patent-Recherche-Agent: Durchsucht Datenbanken und erstellt Berichte
   - Multi-Agenten-Debattiersystem: 3 Agenten bewerten Ideen aus verschiedenen Perspektiven
4. **Technische Architektur** (vereinfacht, aber nicht dumbed down)
   - RAG-Systeme, Vektordatenbanken, Tool-Use, MCP
5. **Kosten und Timeline**
   - Einfacher Agent: 5.000-15.000€, 2-4 Wochen
   - Komplexer Multi-Agent: 15.000-40.000€, 6-12 Wochen
   - Laufende Kosten: 200-800€/Monat (API + Hosting)
6. **FAQ** (featured snippet optimiert)
7. **CTA:** Erstgespräch buchen

### Cluster-Artikel

**Artikel 1: "KI-Agent vs. Chatbot: Der Unterschied, der für Ihr Unternehmen zählt"**
- Hochfrequentes Vergleichs-Keyword
- Klare Abgrenzung mit Beispielen

## Technische SEO-Basics (für Claude Code)

### Must-haves für jede neue Seite:

```
- Eigene URL mit sprechendem Slug
- Title Tag: Primäres Keyword + Benefit (max. 60 Zeichen)
- Meta Description: Mit CTA (max. 155 Zeichen)
- H1: Nur eine pro Seite, enthält Hauptkeyword
- H2/H3: Strukturierte Zwischenüberschriften mit Nebenkeywords
- Interne Links: Mindestens 3 pro Seite (zu Pillars, Use Cases, Blog)
- Schema Markup: FAQPage für FAQ-Sektionen, LocalBusiness für Unternehmensinfo
- Open Graph Tags: Für LinkedIn-Sharing (wichtig für deine Strategie)
- Canonical URL: Jede Seite hat eine kanonische URL
- Sitemap.xml: Automatisch generiert, bei Google Search Console eingereicht
- robots.txt: Erlaubt Crawling aller relevanten Seiten
```

### Next.js-spezifisch:

```
- generateMetadata() für dynamische Title/Description
- generateStaticParams() für statische Generierung der Use-Case-Seiten
- next-sitemap oder manuelles sitemap.ts
- Structured Data als JSON-LD im Head
- Breadcrumbs auf jeder Unterseite
```

### Google Search Console:

```
- Property einrichten für aimation.de (falls nicht bereits geschehen)
- Sitemap einreichen
- URL-Prüfung für jede neue Seite
- Index-Abdeckung überwachen
```

Jede Seite verlinkt auf mindestens 2 andere Seiten. Jede Pillar wird von mindestens 3 anderen Seiten verlinkt.
